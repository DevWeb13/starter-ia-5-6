# Codex local avec VS Code

## Utiliser Codex local pour

- inspecter un dépôt approuvé ;
- modifier des fichiers sur une branche dédiée ;
- exécuter les contrôles locaux autorisés et préparer des commits vérifiables.

## Démarrage

Ouvrez le dépôt dans VS Code, vérifiez la branche et l’arbre Git, puis donnez un brief borné. Dans ce dépôt, Codex applique [`../../AGENTS.md`](../../AGENTS.md) et [`../../WORKFLOW.md`](../../WORKFLOW.md).

## Limites et passage de relais

Les fichiers et processus restent sur la machine. Les actions externes dépendent des permissions et du brief. Pour piloter la même session depuis un iPhone, utilisez [Codex Remote](codex-remote.md) ; pour une préparation cloud séparée, utilisez le workflow [hybride](hybrid-work-codex.md).
