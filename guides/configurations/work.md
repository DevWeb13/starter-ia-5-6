# Work : mission complète dans le cloud

## Utiliser cette configuration pour

- exécuter un brief en plusieurs étapes ;
- analyser des fichiers et coordonner les outils ou plugins réellement disponibles ;
- produire et vérifier un livrable complet dans l’environnement cloud.

## Démarrage en trois étapes maximum

1. Remplissez [`templates/BRIEF.md`](../../templates/BRIEF.md) avec les livrables, limites et critères de réussite.
2. Adaptez [`prompts/MASTER-WORK.md`](../../prompts/MASTER-WORK.md) à la mission.
3. Relisez le livrable réel avec [`QUALITY.md`](../../QUALITY.md).

## Ressources associées

- [`WORKFLOW.md`](../../WORKFLOW.md) pour le processus de référence ;
- [`prompts/REVIEW.md`](../../prompts/REVIEW.md) pour une revue indépendante ;
- [`prompts/AUTOMATION.md`](../../prompts/AUTOMATION.md) pour un audit GitHub prudent si cette fonction est disponible ;
- [`course/FORMATION-EXPRESS.md`](../../course/FORMATION-EXPRESS.md) et [`START-HERE.md`](../../START-HERE.md) pour le parcours d’apprentissage sur iPhone.

## Limites

Work ne charge pas automatiquement `AGENTS.md` ni `.codex/config.toml`. Les outils, plugins, modèles et automatisations dépendent du compte et de l’environnement ; vérifiez leur présence avant de les inclure dans le brief.

## Passage de relais

Pour modifier et vérifier un dépôt sur la machine de développement, transmettez le brief, les sources, les décisions et les limites à [Codex local](codex-local.md), directement ou avec le guide [Work + Codex](hybrid-work-codex.md).
