# Workflow hybride Work + Codex

## Utiliser ce workflow pour

- préparer ou analyser une mission complète dans Work ;
- conserver l’implémentation, Git et les tests sur la machine avec Codex ;
- séparer clairement décisions, écriture et vérification.

## Passage de relais

1. Dans Work, produire un brief, les décisions, les sources et les critères de réussite.
2. Transmettre à Codex uniquement ce paquet de relais, sans secret.
3. Dans Codex, vérifier le dépôt, la branche, `AGENTS.md` et l’état existant avant d’écrire.
4. Implémenter avec un seul écrivain, exécuter les contrôles autorisés et rendre les preuves.
5. Rapporter à Work le résultat réel, les commits locaux et les blocages, sans supposer une publication.

[`../../WORKFLOW.md`](../../WORKFLOW.md) reste la source de vérité commune. Work ne charge pas automatiquement les règles locales du dépôt ; le paquet de relais doit donc mentionner les contraintes qui influencent la mission.
